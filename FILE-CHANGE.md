# การทำบันทึกการเข้าถึงไฟล์ (File Auditing) ใน auditd

การทำบันทึกการเข้าถึงไฟล์ (File Auditing) ใน auditd จะใช้วิธีที่เรียกว่า File System Watch เพื่อดักจับพฤติกรรม เช่น การอ่าน การเขียน หรือการแก้ไขสิทธิ์ของไฟล์ที่ระบุ โดยมีขั้นตอนการทำดังนี้
## 1. วิธีเพิ่มกฎแบบชั่วคราว (เห็นผลทันที แต่หายเมื่อรีสตาร์ท)
ใช้คำสั่ง auditctl เพื่อทดสอบกฎก่อนใช้งานจริง:

sudo auditctl -w /path/to/file -p rwxa -k file_monitor

## 2. วิธีเพิ่มกฎแบบถาวร (แนะนำสำหรับการใช้งานจริง)
หากต้องการให้กฎยังคงอยู่หลังจากรีสตาร์ทระบบ ให้ทำตามขั้นตอนนี้:

   1. เปิดไฟล์กฎคอนฟิก:
   
   sudo nano /etc/audit/rules.d/audit.rules
   
   2. เพิ่มบรรทัดคำสั่งไว้ด้านล่างสุดของไฟล์:
   
   -w /etc/shadow -p wa -k shadow_changes
   
   3. ความหมายของพารามิเตอร์:
   * -w : ระบุพาร์ทของไฟล์หรือไดเรกทอรีที่ต้องการเฝ้าดู
      * -p : ระบุสิทธิ์ (Permissions) ที่ต้องการดักจับ เลือกผสมกันได้จาก:
      * r = Read (การอ่านไฟล์)
         * w = Write (การเขียนหรือแก้ไขไฟล์)
         * x = Execute (การสั่งรันไฟล์)
         * a = Attribute (การเปลี่ยนสิทธิ์/เจ้าของไฟล์ เช่น chmod, chown)
      * -k : กำหนดคำค้นหา (Key) เพื่อใช้ฟิลเตอร์ดูในล็อกภายหลัง (ตั้งชื่ออะไรก็ได้)
   4. รีสตาร์ทบริการเพื่อให้กฎทำงาน:
   
   sudo systemctl restart auditd
   
   
## 3. วิธีตรวจสอบและค้นหาบันทึก
เมื่อมีคนเข้าถึงไฟล์ตามเงื่อนไข ระบบจะบันทึกลงไฟล์ /var/log/audit/audit.log คุณสามารถใช้คำสั่ง ausearch ค้นหาผ่านคีย์เวิร์ดที่ตั้งไว้ได้ทันที:

sudo ausearch -k shadow_changes

หากต้องการค้นหาเจาะจงเฉพาะไฟล์ สามารถระบุพาร์ทตรงๆ ได้เช่นกัน:

sudo ausearch -f /etc/shadow

------------------------------
หากต้องการให้แนะนำเพิ่มเติม แจ้งได้เลยนะ:

* ต้องการเฝ้าดู ไฟล์ระบบทั่วไป หรือ ไดเรกทอรีขนาดใหญ่?
* ต้องการส่งล็อกนี้ไปยัง ระบบศูนย์กลาง (Centralized Log/SIEM) หรือไม่?
* ต้องการ คัดกรองยูสเซอร์บางคน (Exclude) ออกจากการตรวจจับไหม?
